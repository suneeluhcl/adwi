# Bounded Self-Upgrade Control

The workspace may improve its own prompts, reports, scripts, indexes, and routing policies when the change is local, inspectable, reversible, and inside `~/SuneelWorkSpace`.

## Allowed Autonomous Actions

- Refresh system audit, gap analysis, recommendations, and system profile metadata.
- Create plain-file research plans, analyses, decision drafts, and improvement plans.
- Add or update local wrapper commands that call existing workspace scripts.
- Update MCP resource maps for new durable workspace artifacts.
- Update shared memory, decisions, insights, task state, and logs after meaningful work.

## Requires Explicit Approval

- Installing external tools or plugins.
- Spending money, changing subscriptions, or modifying billing.
- Sending email/messages or contacting people.
- Deleting, moving, or bulk-renaming private files.
- Deep-scanning private folders outside the workspace.
- Running destructive git, filesystem, database, or app automation commands.

## Rollback Expectations

- Important file replacements need timestamped backups.
- Autolab experiments must keep rollback notes or snapshots.
- New commands should be small, inspectable, and backed by plain files.
