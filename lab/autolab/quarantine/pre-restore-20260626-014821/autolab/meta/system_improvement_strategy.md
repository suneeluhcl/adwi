# System Improvement Strategy

Autolab should use `audit/gap_analysis.md`, `audit/improvement_plan.md`, `tools/recommendations.md`, and `agent-system/shared/BOUNDED_SELF_UPGRADE.md` as inputs when proposing workspace self-improvements.

## Evaluation Signals

- System audit exists and is current.
- Gap analysis classifies architecture, automation, intelligence, research, workflow, and usability gaps.
- Tool inventory is current and marks safe integration status.
- Research-engine can capture ideas and produce analysis files.
- MCP exposes new audit, research, profile, and tool resources.

## Mutation Limits

- Do not install external tools.
- Do not spend money or change accounts.
- Do not scan private file contents outside the workspace without explicit request.
- Keep every improvement local-first, inspectable, and reversible.
